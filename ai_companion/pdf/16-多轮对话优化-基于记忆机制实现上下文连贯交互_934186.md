一、多轮对话的本质：从“即时响应”到“持续理解”

在传统软件开发中，用户请求通常是独立且无状态的。比如你在电商网站点击“加入购物车”，系统完成操作后并不需要记住你之前浏览了哪些商品。但在情感聊天场景中，每一次对话都是前一次的延续，用户的每一句话都承载着情绪积累、背景信息和潜在期待。

多轮对话的核心目标，是让AI具备“对话连续性”（Conversational Continuity）。这意味着：

语义连贯：AI能理解当前话语在整体对话流中的位置。

上下文感知：AI能调用历史信息解释当前输入。

情感延续：AI能感知情绪变化并做出适配回应。

目标导向：AI能在长期互动中维持特定角色或任务目标。

2.1 短期记忆：滑动窗口与关键轮次保留

短期记忆对应大模型的上下文窗口。我们采用“滑动窗口 + 重点保留”策略。

滑动窗口：保留最近5~10轮完整对话，确保近期上下文完整。

关键轮次保留：标记并强制保留具有重要意义的对话轮次，如：

用户首次表达核心困扰（“我最近总是失眠”）

情绪转折点（“其实我已经辞职了”）

明确请求（“你能帮我列个计划吗？”）

2.2 长期记忆：向量数据库的语义检索

当对话跨度超过几天甚至几周时，仅靠上下文窗口无法满足需求。此时需依赖向量数据库进行长期记忆存储与检索。

我们在第十讲已搭建了向量数据库（如Chroma或Pinecone），现在进一步优化其使用方式。

记忆写入：每次对话结束后，提取关键信息生成记忆片段并存入向量库。

记忆检索：在新对话开始时，根据当前输入语义检索最相关的过往记忆。

检索策略：

多跳检索：支持“由点及面”的联想式回忆，如从“失眠”联想到“工作压力”。

3.2 记忆质量评估与优化

为确保记忆系统有效，我们引入两个评估维度。

相关性评分：人工标注历史记忆与当前输入的相关性（0~5分），定期评估检索准确率。

```python
# backend/services/memory_quality_evaluator.py

class MemoryQualityEvaluator:
    """记忆质量评估器"""

    async def calculate_relevance_score(
        self,
        retrieved_memories: List[Dict],
        current_query: str
    ) -> Dict[str, float]:
        """
        计算相关性评分
        返回：{
            "avg_relevance": 4.2,  # 平均相关性（0-5分）
            "high_relevance_ratio": 0.75,  # 高相关性（≥4分）占比
            "recall_rate": 0.82  # 召回率
        }
        """
        # 1. 使用LLM评估每个记忆与当前查询的相关性
        relevance_scores = []
        for memory in retrieved_memories:
            score = await self._llm_evaluate_relevance(
                memory["content"],
                current_query
            )
            relevance_scores.append(score)

        avg_relevance = sum(relevance_scores) / len(relevance_scores)
        high_relevance_ratio = sum(1 for s in relevance_scores if s >= 4) / len(relevance_scores)

        # 2. 计算召回率（需要人工标注的"应召回记忆"作为ground truth）
        recall_rate = await self._calculate_recall_rate(retrieved_memories, current_query)

        return {
            "avg_relevance": avg_relevance,
            "high_relevance_ratio": high_relevance_ratio,
            "recall_rate": recall_rate
        }

    async def calculate_user_satisfaction(
        self,
        user_id: str,
        days: int = 7
    ) -> Dict[str, float]:
        """
        计算用户满意度
        基于用户反馈数据统计
        """
        feedbacks = await self._get_user_feedbacks(user_id, days)

        if not feedbacks:
            return {"satisfaction_rate": 0.0, "total_feedbacks": 0}

        positive_count = sum(1 for f in feedbacks if f["rating"] >= 4)
        satisfaction_rate = positive_count / len(feedbacks)

        return {
            "satisfaction_rate": satisfaction_rate,
            "total_feedbacks": len(feedbacks),
            "positive_count": positive_count
        }

    async def generate_quality_report(
        self,
        user_id: Optional[str] = None,
        date_range: Tuple[datetime, datetime] = None
    ) -> Dict[str, Any]:
        """
        生成记忆质量评估报告
        最终落地结果：结构化的评估报告
        """
        # 1. 检索准确率
        relevance_metrics = await self._calculate_batch_relevance(date_range)

        # 2. 用户满意度
        satisfaction_metrics = await self._calculate_batch_satisfaction(user_id, date_range)

        # 3. 记忆利用率
        utilization_rate = await self._calculate_memory_utilization(date_range)

        # 4. 对话连贯性（需要人工抽样评估）
        coherence_score = await self._evaluate_conversation_coherence(date_range)

        # 5. 综合评分
        overall_score = (
            relevance_metrics["high_relevance_ratio"] * 0.3 +
            satisfaction_metrics["satisfaction_rate"] * 0.3 +
            utilization_rate * 0.2 +
            coherence_score * 0.2
        )

        return {
            "report_date": datetime.now().isoformat(),
            "overall_score": overall_score,  # 0-1分，综合质量评分
            "relevance_metrics": relevance_metrics,
            "satisfaction_metrics": satisfaction_metrics,
            "utilization_rate": utilization_rate,
            "coherence_score": coherence_score,
            "recommendations": self._generate_recommendations(overall_score, {
                "relevance": relevance_metrics,
                "satisfaction": satisfaction_metrics
            })
        }
```

3.2.2 记忆重要性加权计算

记忆的最终重要性分数由多个因子综合计算。

```python
def calculate_memory_importance(
    memory: Dict[str, Any],
    base_similarity_score: float,  # 向量检索的相似度分数（0-1）
    days_ago: int,
    access_count: int
) -> float:
    """
    计算记忆的最终重要性分数
    用于排序和筛选
    """
    # 1. 基础相似度
    importance = base_similarity_score

    # 2. 记忆类型权重
    type_weights = {
        "commitment": 1.8,
        "relationship": 1.6,
        "event": 1.4,
        "concern": 1.5,
        "preference": 1.3
    }
    importance *= type_weights.get(memory.get("type", "event"), 1.0)

    # 3. 情绪强度加成
    emotion_intensity = memory.get("emotion_intensity", 5.0)
    if emotion_intensity > 7.5:
        importance *= 1.5  # +50%
    elif emotion_intensity < 5.0:
        importance *= 0.7  # -30%

    # 4. 访问频率加成
    access_bonus = min(access_count * 0.05, 0.5)  # 最多+50%
    importance *= (1 + access_bonus)

    # 5. 时间衰减
    decay_factor = 0.9 ** days_ago  # 每天衰减10%
    importance *= decay_factor

    return importance
```

自动优化检索参数

如果检索准确率<70%，降低相似度阈值，提高召回。

如果召回率<60%，增加检索数量（top_k从5增加到10）。

根据用户反馈，自动调整记忆类型权重。例如：如果“承诺类”记忆的满意度高，则将其权重从1.8x提升到2.0x。

1. 记忆衰减机制

```python
def decay_score(score, days_ago):
    return score * (0.9 ** days_ago)  # 每天衰减10%
```

这需要结合情绪趋势分析与事件关联推理，实现“有温度的主动记忆”。

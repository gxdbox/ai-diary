1.1 大模型的“情感表达能力”从何而来？

Science 2002年11月发表题为《Emotion，cognition，and behavior》的论文，论文中指出，情感认知能力在人类社会中泛指感知、识别和理解他人情感的能力。但受限于对人机交互机理的认知不够清晰，在计算机视域下该能力收敛于“情感计算”。该概念由麻省理工学院媒体实验室Picard于1997年提出，旨在以计算为手段，研究和开发能够识别、解释和处理人类情感系统的一个跨学科领域，涉及计算机科学、心理学和认知科学。

情感计算的基础是情感量化模型，主要用于将无形的、抽象的人类情感转化为机器人可以理解、表达和处理的数据形式。ChatGPT等基于Transformer结构的大语言模型语义理解能力的阶跃式提升对单模态情感认知产生了冲击。大模型本身并不具备情感，但它通过海量人类对话数据的学习，掌握了情感表达的语言模式。例如：

安慰语句常包含“我理解”“你并不孤单”“慢慢来”等词汇。

鼓励语句多使用积极动词和肯定结构，如“你可以的”“我相信你”。

共情表达倾向于使用感受类词汇，如“委屈”“疲惫”“无助”。

这些模式构成了AI“拟人化表达”的基础。但问题在于：模型不知道什么时候该用哪种模式。这正是开发者需要介入的地方。

1.2 响应生成的三大挑战

在人文主义传统中，人拥有情感是一种自明的自我理解。人是情感的存在物，情感则是人对对象的感受和体验。发展到人机传播，人的感受和体验将与人工智能的理解、生成和回应情感的能力紧密相关。机器通过与用户的长期互动，逐步形成准确识别、理解用户情感的能力，生成带有情感色彩的文本、语音等多模态内容，人机之间进而产生深层次情感关系。

如图12-1所示，人类与自己创造出的无意识但又极为强大的非人类智能共存时，必须时刻审视向内的心灵成长和内向超越，自我修正情感的偏执、茧化、极化，甚至是疯狂。在人机交流的实践中，关心和提升人类大脑、身、心、灵的整合发展，找到心之所归。

2.3.1 规则引擎：保障安全底线

对于检测到“自残”、“抑郁重度”等高风险意图，不依赖LLM生成，而是直接触发预设的危机干预话术，并建议用户联系专业机构。

```python
def _is_crisis_situation(self, user_emotion: str, metadata: Optional[Dict]) -> bool:
    """判断是否为危机情况（核心逻辑）"""
    if user_emotion == "high_risk_depression":
        return True
    if metadata and metadata.get("requires_crisis_intervention"):
        return True
    return False

def _handle_crisis(self, user_input: str, user_emotion: str, metadata: Optional[Dict]) -> str:
    """处理危机情况，返回预设的危机干预回复（核心逻辑）"""
    crisis_strategy = self.emotion_strategy.get("high_risk_depression", {})
    crisis_response = crisis_strategy.get("fallback", "")

    if crisis_response:
        return crisis_response

    # 默认危机回复（核心模板）
    return """我非常关心你现在的情绪状态。你不是一个人，有很多人愿意帮助你。
建议你立即联系心理援助热线：
- 希望24热线：400-161-9995
- 北京心理危机干预中心：010-82951332
我会一直在这里陪你。"""
```

2.3.2 缓存匹配：提升响应效率

对高频、固定场景（如问候、道别），使用键值对缓存，避免重复调用大模型：

```python
def _match_cached_response(self, user_input: str, user_emotion: str) -> Optional[str]:
    """匹配缓存的固定回复"""
    input_lower = user_input.lower().strip()

    # 问候语
    greetings = ["你好", "hi", "hello", "嗨"]
    if any(g in input_lower for g in greetings) and len(input_lower) < 10:
        return random.choice(self.cached_responses.get("greeting", []))

    # 道别语
    farewells = ["再见", "拜拜", "bye"]
    if any(f in input_lower for f in farewells):
        return random.choice(self.cached_responses.get("goodbye", []))

    return None
```

```python
def _call_llm(self, prompt: str) -> str:
    """调用大模型生成回复"""
    # 兼容不同的LLM客户端接口
    if hasattr(self.llm_client, 'generate'):
        response = self.llm_client.generate(prompt)
    elif hasattr(self.llm_client, 'predict'):
        response = self.llm_client.predict(prompt)
    elif hasattr(self.llm_client, 'invoke'):
        response = self.llm_client.invoke(prompt)

    # 提取文本内容
    if hasattr(response, 'content'):
        response = response.content
    elif isinstance(response, dict) and 'content' in response:
        response = response['content']

    return str(response).strip()
```

三、情感一致性校验：让AI“说到做到”

即使我们精心设计了提示词，大模型仍可能“跑偏”。例如：

用户处于悲伤情绪，AI却回复“哈哈，这有什么好难过的”。

连续对话中，AI突然改变语气或角色设定。

为此，我们需要引入情感一致性校验机制。

3.1 基于小模型的情绪分类器

我们可以训练一个轻量级情绪分类器（如BERT微调版），对AI生成的回复进行二次检测，判断其情绪倾向是否与用户输入匹配。

```python
ef check_emotion_consistency(self, ai_response: str, user_emotion: str) -> Tuple[bool, str]:
    """检查AI回复的情绪与用户情绪是否一致"""
    # 1. 检测AI回复的情绪
    response_emotion, confidence = self.detect_emotion(ai_response)

    # 2. 检查是否是严重不兼容的组合
    forbidden_combinations = [
        ("sad", "happy"),
        ("anxious", "excited"),
        ("high_risk_depression", "happy")
    ]
    for forbidden in forbidden_combinations:
        if (user_emotion in forbidden and response_emotion in forbidden):
            return False, f"严重情感错配：用户{user_emotion}，AI回复{response_emotion}"

    # 3. 检查兼容性
    compatible_emotions = self.EMOTION_COMPATIBILITY.get(user_emotion, ["neutral"])
    if response_emotion in compatible_emotions:
        return True, "情感匹配"

    return False, f"情感不匹配：用户{user_emotion}，AI回复{response_emotion}"
```

完整实现请参考：backend/utils/sentiment_classifier.py

若不一致，则触发降级策略，使用更安全的预设回复，或重新生成。

3.2 角色稳定性监控

通过关键词监控和语义相似度计算，确保AI不偏离预设角色。例如：

禁止使用“我是GPT”“我是一个AI”等暴露身份的表述；

避免突然切换语气（如从温柔变为严肃）。

```python
def _post_process_response(self, response: str, user_emotion: str) -> str:
    """后处理生成的回复"""
    # 确保不暴露AI身份
    identity_replacements = {
        "我是AI": "我是心语",
        "我是一个AI": "我是心语",
        "作为AI": "作为陪伴者"
    }
    for old, new in identity_replacements.items():
        if old in response:
            response = response.replace(old, new)

    return response
```

可在日志中记录每次回复的“角色偏离度”，用于后续优化。

4.2 核心代码实现

```python
class ResponseGenerator:
    """响应生成器 - 混合架构"""

    def __init__(self, llm_client, strategy_file="config/emotion_strategy.yaml"):
        self.llm_client = llm_client
        with open(strategy_file, 'r', encoding='utf-8') as f:
            self.strategy = yaml.safe_load(f)
        self.prompt_builder = DynamicPromptBuilder(self.strategy)
        self.sentiment_classifier = SentimentClassifier()

    def generate_response(self, user_input, user_emotion, user_id, emotion_intensity=5.0):
        """生成AI回复（主入口）"""
        # 1. 检查是否为高风险情况（危机干预）
        if self._is_crisis_situation(user_emotion, None):
            return self._handle_crisis(user_input, user_emotion, None)

        # 2. 缓存匹配（高频固定场景）
        cached_response = self._match_cached_response(user_input, user_emotion)
        if cached_response:
            return cached_response

        # 3. LLM生成（主要路径）
        prompt = self.prompt_builder.build_prompt(
            user_input=user_input,
            emotion=user_emotion,
            emotion_intensity=emotion_intensity
        )

        raw_response = self._call_llm(prompt)
        processed_response = self._post_process_response(raw_response, user_emotion)

        # 4. 情感一致性校验
        is_valid, warnings = self.sentiment_classifier.comprehensive_check(
            processed_response, user_emotion
        )

        if not is_valid:
            # 降级为预设回复
            return self._get_fallback_response(user_emotion)

        return processed_response
```
